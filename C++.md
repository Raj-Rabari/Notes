1. #include <iostream>
   This is not some import statement like javascript or python. "#include" is preprocessor directive. before compiler compiles the code preprocessor like this will copy the code of libraries in your file(imp: they just copy the .h header files means the declarations so that at compile time compiler will know that these things exist and linker will link their implementation(binary code) later on the process.)

In higher level environment like python, java or javascript. When you declare a variable, engine handle everything. you say i need a box, engine finds the box in ram, sizes it, put data in it and clean it later.

2. Variables :
   Imagine your RAM as lakhs of empty boxes of 1 byte. Every box has unique identfier called memory address (mostly represented in hexa decimal). when you write int age = 25; what happens under the hood is that,
   1. program requests OS that i need some memory to store an integer.
   2. now OS knows that in c++ integer require 4 bytes of data so it finds 4 continues boxes(let say 0x0001,0x0002,0x0003 and 0x0004).
   3. compiler internally map the name age to the initial address(0x0001). The CPU doesn't know what age is it just knows to look at the memory address (0x0001).
   4. The number 25 converted to binary and shoved into those 4 boxes.

differnce datatype require different amounts of memory, like char needs 1 byte and double(more precise decimal number) needs 8 bytes.

& operator :
using this variable you can get the physical adress of memory.

Program execution flow:

    1. Preprocessor: It doesn't know c++. It just a text manipulation tool. If it sees text starting with # (#include<iostream> or #define PI 3.14) it copies the code if it's library or copies the value if it's macro, etc. It also remove comments as machine doesn't need our notes.
    2. now compiler takes over the output source code generated by preprocessor and checks for syntax error(like missing semicolon or type mismatch etc). If everything goes well then it converts your code to assembly language(low level language specific to CPU architecture like x64 or ARM). the code looks like instructions to CPU about which registers to move memory into(MOV, ADD, PUSH etc).
    3. Assembler takes the assembly language code generated above and convert it to binary 0s and 1s which computer can understand. It still not ready to run. Because if you use any library like iostream. preprocessor just copy the header files(which contains declarations) in source code. The actucal implementation of it is in compiled binary form which is linked by linker at next step.
    4. Linker links the binary implementaion of libarries you used in your code.

Pointer Variable:
We can get physical address using the & operator but we can't store them for later user in normal variables. Memory address is a special type of data and you will need pointer variables to store the address in variables.

There are two faces of "\*" :

    while creating variable if you use pointer next to the data type like int_ , you are saying that this is a pointer variable and It will store memory adress of integer type of variable.

    if you use * on existing pointer, program says to CPU that look at the adress stored inside this pointer variable and travel to that adress in ram and give me the value stored at that address.

Here are some use cases in which you can see use of pointer:

    Dynamic memory: Sometimes you don't know how much amount of memory you will need until the programm is already running(like reading user input or downloading data from network). pointer allows you to ask OS for more memory on the fly and tracks where OS puts it.

    Passing giant files or large data:

        For data of data type int or char, it won't make any sense of 64 bit OS . because int data takes 4 byte and char needs 4 bytes. while on 64 bit system memory address is too long and require 8 bytes.

        But for large data it will very useful. here is how: As execution of any function starts c++ engine creates brand new execution context or stack memory area for that function to store it's args and local variables. when you call the function, C++ copies the data in the memory of that function. So now you can't change the data outside of this function and copying large data will be expensive. So here pointer comes in picture. you can pass pointer variables to the function, so now you can change the value stored in that memory address. And when the function execution starts it only copies the pointer variable (the address) in function's stack frame so the big data (file of 2GB) won't get copied again in the function and we can save here more time and memory.
