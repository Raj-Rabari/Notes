stack memory: It's the part of total memory space available for program to run. It's small part that is designed for fast access to the memory. and stores the data as in stack data structure in LIFO philosophy. All the data stored on stack must have known , fixed size. if any data that has size unkwown at compile time or might change in runtime must be stored in heap area.

Heap: It is less organized than stack, when you request a memory, memory allocator finds an empty spot that is big enough, mark it as being in use and returns pointer(address of that memory location).
